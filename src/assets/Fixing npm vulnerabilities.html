To check which port your local server is running on in Windows, here are a few easy ways — pick the one that works best for your situation:

🔎 Option 1: Check Your Terminal Output

If you're running a dev server (like npm start, vite, webpack, etc.), it usually prints the port in the terminal, like:

arduino
Local: http://localhost:3000/

💻 Option 2: Use netstat or PowerShell

Command Prompt:

bash
netstat -aon | findstr :<port>

Replace <port> with the number you're checking (e.g., 3000):

bash
netstat -aon | findstr :3000

You’ll see output like:

nginx
  TCP    127.0.0.1:3000     0.0.0.0:0     LISTENING     12345

The last number is the Process ID (PID).

🔧 Option 3: Match PID to App

Once you have the PID (e.g., 12345), find what app is using it:

Command Prompt:

bash
tasklist | findstr 12345

Or in PowerShell:

powershell
Get-Process -Id 12345

🌐 Option 4: Use Resource Monitor (GUI)

  1. Open Task Manager (Ctrl + Shift + Esc)

  2. Go to Performance tab

  3. Click Open Resource Monitor at the bottom

  4. Go to Network > Listening Ports

  5. Look for the Port, PID, and Process Name

If you're just trying to find where your Node/Vite/React server is running, your terminal usually tells you. But if you're debugging something deeper (like conflicts or finding stray servers), netstat or Resource Monitor is the way to go.

In recent versions of react-date-range, the locale prop is required if internal default config is missing or improperly bundled. When it’s missing, it tries to use undefined.localize, causing that error.
